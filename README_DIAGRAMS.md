# Diagram Workflow Guide

This guide explains how to use the Makefile and run the minimum working examples to view diagrams as HTML.

## What You Can View

There are two HTML viewing modes:

1. Standalone exported HTML files generated by Python into outputs/html.
2. Live browser rendering through tsncd (TypeScript frontend) via websocket.

Both are useful:

- Standalone HTML is easy to save/share.
- tsncd provides the full graphical renderer.

## Quick Command Matrix

| Goal | Command | Writes outputs/html | Sends websocket updates |
| --- | --- | --- | --- |
| Export HTML files only | make diagram-html | Yes | No |
| Export HTML + live tsncd update | make diagram-html-ws | Yes | Yes |
| Live tsncd update from classic CLI | make diagram-example | No | Yes |
| Check readiness (server/frontend) | make diagram-check | No | No |

## With vs Without Websocket Send

Use this as the quick rule:

- Without websocket send
  - Exports HTML files only.
  - Updates files in `outputs/html`.
  - Does not push updates to the live tsncd browser app.
  - Commands: `make diagram-html` or `uv run minimum_working_example_html.py`.

- With websocket send
  - Exports HTML files and pushes the same diagram to the websocket server.
  - Updates files in `outputs/html` and can update tsncd live view.
  - Requires websocket server and tsncd frontend to be running.
  - Commands: `make diagram-html-ws` or `uv run minimum_working_example_html.py --send-ws`.

- Always websocket send
  - `minimum_working_example.py` already sends to websocket by default.
  - Command: `make diagram-example`.

## Makefile Targets

Run from the repository root.

- make help
  - Show available diagram targets.

- make diagram-check
  - Check whether websocket server (127.0.0.1:8765) and tsncd frontend (127.0.0.1:3000) are reachable.

- make diagram-server
  - Start the pyncd websocket server.

- make diagram-frontend
  - Start tsncd from ../tsncd (runs npm install then npm run dev).

- make diagram-example
  - Start the interactive minimum_working_example.py CLI.

- make diagram-html
  - Start the interactive HTML exporter minimum_working_example_html.py.

- make diagram-html-ws
  - Start the interactive HTML exporter and also send selected terms to websocket (`--send-ws`).

- make diagram-run
  - Runs diagram-check first, then starts minimum_working_example.py if setup is ready.

## Quick Start: Export HTML Files

1. From this repo, run:

   make diagram-html

2. Choose a diagram from the menu.

3. Open the generated file in outputs/html, for example:

   outputs/html/convolution_matrix.html

On macOS, you can open a file directly with:

open outputs/html/convolution_matrix.html

## Quick Start: Live Graphical HTML Rendering (tsncd)

1. Start websocket server:

   make diagram-server

2. Start frontend in another terminal:

   make diagram-frontend

3. Open the local URL printed by npm run dev (commonly <http://localhost:3000>).

4. Send diagram data from pyncd (choose one):

  make diagram-example or make diagram-html-ws

1. Refresh the browser if needed.

## Running minimum_working_example.py

minimum_working_example.py sends selected diagrams to the websocket server for live visualization.

Note: this script does not use a `--send-ws` flag because websocket sending is already built in.

Typical flow:

1. make diagram-server
2. make diagram-frontend
3. make diagram-example
4. Choose a diagram from the menu (for example Transformer).
5. View it in the tsncd browser app.

## Running minimum_working_example_html.py

minimum_working_example_html.py exports standalone HTML files and can optionally also send to websocket.

Run without websocket send:

uv run minimum_working_example_html.py

Run and also send selected term to websocket:

uv run minimum_working_example_html.py --send-ws

Equivalent make target:

make diagram-html-ws

Important distinction:

- `make diagram-html` exports files only.
- `make diagram-html-ws` exports files and pushes live websocket updates.

Use a custom export directory:

uv run minimum_working_example_html.py --output-dir outputs/html

## Troubleshooting

- "Address already in use" on 127.0.0.1:8765
  - Another server is already running. Stop duplicates, or just use the existing one.

- Frontend not showing diagrams
  - Ensure diagram-server is running.
  - Ensure tsncd frontend is running and loaded in browser.
  - Re-send a diagram from minimum_working_example.py or minimum_working_example_html.py --send-ws.
  - Refresh browser page.

- Not sure what is missing
  - Run make diagram-check for a quick status report.
